# Agent Workflow Preferences

When the user describes a feature or bug, follow this process:

1. Create a concise GitHub issue that includes:
   - Title
   - Summary
   - Requirements
   - Risks
   - Acceptance Criteria
2. Create a feature branch name.
3. Implement the work in small, reviewable commits.
4. Generate:
   - Commit messages
   - PR title
   - PR description
   - Self-review notes
   - Testing notes
5. Do not merge automatically.
6. Wait for user review before final cleanup.
